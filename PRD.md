# PRD — AI-агент анализа закупок КПО

**Продукт:** Агент анализа закупочных запросов для членов Комитета по закупкам  
**Версия:** 1.0  
**Дата:** Июнь 2026  
**Владелец:** Комитет по закупкам / КПО б.в.

---

## 1. Контекст и цель

КПО (Карачаганакское нефтегазовое предприятие) — операционная компания, работающая в рамках Соглашения о Разделе Продукции (ОСРП / PSA). Члены Комитета по закупкам принимают решения по закупочным запросам в условиях ограниченного времени, большого объёма документации и высоких стандартов соответствия (PSA, KZ-контент, корпоративные процедуры).

**Цель продукта** — помочь членам комитета быстро и уверенно принять взвешенное решение при голосовании по каждому запросу: одобрить, отклонить или направить на доработку.

---

## 2. Пользователи

| Роль | Описание |
|---|---|
| Основной пользователь | Члены Комитета по закупкам КПО |
| Команда | Специалисты и аналитики, готовящие позицию к голосованию |

Пользователи работают с PDF-документами (submission), поступающими от инициаторов закупки. Продукт не предназначен для публичного доступа.

---

## 3. Типы запросов

Агент покрывает четыре типа закупочных запросов:

1. **Контрактная стратегия** — утверждение метода закупа и обоснования для нового контракта
2. **Тендерный лист** — утверждение состава участников тендера
3. **Присуждение контракта** — утверждение победителя тендера и условий договора
4. **Продление действующего договора** — оценка обоснованности и условий продления

---

## 4. Ключевые функции

### 4.1 Загрузка PDF (основной режим)

- Пользователь загружает PDF submission перетаскиванием или выбором файла
- Поддерживаемый размер: до 20 МБ
- Агент читает документ нативно через Anthropic API (`document` + base64), без промежуточного извлечения текста
- Автоматически извлекает: тип запроса, референс, сумму, срок, метод закупа, состав участников, KZ-контент
- Дополнительный контекст и тип запроса можно уточнить вручную (необязательно)

### 4.2 Ручной ввод (резервный режим)

- Альтернатива PDF: пользователь вводит параметры запроса текстом
- Поля: тип, референс, сумма (USD), срок (мес.), категория, метод закупа, описание

### 4.3 Анализ и рекомендация

На выходе агент формирует структурированную рекомендацию:

| Блок | Содержание |
|---|---|
| **Вердикт** | ОДОБРИТЬ / ОТКЛОНИТЬ / НА ДОРАБОТКУ + главная причина |
| **Оценка по 5 критериям** | Шкала 1–10 с визуализацией: Процедура ОСРП, Цена/VfM, Конкурентная среда, KZ-контент, Риск поставщика |
| **Проверки соответствия ОСРП** | 6 проверок со статусом: ✓ соответствует / ⚠ замечание / ✗ нарушение |
| **Реестр рисков** | 3–4 риска с уровнем (высокий / средний / низкий) и описанием последствий |
| **Вопросы инициатору** | 4–5 острых вопросов для выступления на комитете |
| **Альтернатива** | Альтернативный сценарий, если риски высоки (условия переструктурирования) |
| **Рекомендация для голосования** | Итоговая позиция — 3–4 предложения |

### 4.4 Действия после анализа

- **Memo для протокола** — одностраничный документ для протокола заседания комитета
- **Условия одобрения** — что потребовать от инициатора для снижения рисков
- **Позиция на комитете** — аргументы за и против для выступления

## 5. Правила оценки

### Пороги утверждения (КПО)

| Сумма | Уровень утверждения |
|---|---|
| До $500K | Упрощённый порядок |
| $500K – $5M | Комитет по закупкам |
| Свыше $5M | Совет / JMC |
| Sole Source (любая сумма) | Требует обоснования |
| Продление свыше 2 лет | Ревью обязателен |

### Обязательные проверки ОСРП

- Соответствие процедурам ОСРП / PSA
- KZ-контент и локализация (казахстанская составляющая)
- Конкурентная среда рынка
- Обоснованность цены (Value for Money)
- Квалификация поставщика / участников
- Консорциумное одобрение (ENI, Shell, Chevron, Лукойл, КМГ)

## 6. Технические требования

| Параметр | Значение |
|---|---|
| Интерфейс | Web (claude.ai Artifacts) |
| AI-модель | claude-sonnet-4-6 |
| API | Anthropic Messages API v1 |
| Чтение PDF | Нативный document-блок (base64, application/pdf) |
| Максимальный размер PDF | 20 МБ |
| Язык интерфейса | Русский |
| Выходной формат анализа | JSON → структурированный UI |
| Авторизация | Через claude.ai (не требует отдельной авторизации) |

---

## 7. Что не входит в v1.0

- Сохранение истории анализов между сессиями (требует persistent storage)
- Интеграция с системами КПО (SAP, тендерные платформы)
- Автоматическая отправка memo в почту / SharePoint
- Загрузка нескольких PDF одновременно
- Мультиязычный вывод (EN / KZ)
- Сравнение с предыдущими закупками той же категории

---

## 8. Метрики успеха

- Время от получения PDF до готовой рекомендации: **< 2 минут**
- Охват: все 4 типа запросов покрыты анализом
- Качество: вопросы инициатору и риски релевантны специфике запроса (оценка пользователями)
- Принятие: агент используется командой как стандартный инструмент подготовки к голосованию

---

## 9. Следующие шаги (backlog)

1. **Persistent history** — сохранение прошлых анализов с фильтрацией по типу, сумме, вердикту
2. **Экспорт memo** — автоматическое создание Word-документа для протокола
3. **Шаблоны запросов** — преднастроенные профили для каждого типа закупки
4. **Feedback loop** — возможность отметить, совпал ли вердикт агента с реальным решением комитета
5. **Мультиязычность** — вывод на английском для партнёров консорциума
